# 📻 みんなで勝ち時ラジオ｜更新マニュアル（社長用）

録音した音声を、Claude Code に貼るだけで配信できます。
配るフィードURL（社員用）はずっと変わりません：
**https://mediaaidradio-lab.github.io/kachidoki-radio/podcast.xml**

---

## ■ Part 1：初期セットアップ（最初の1回だけ）

> このパートは「最初の1回だけ」の準備です。順番どおりに進めれば一人でできます。
> 所要時間：だいたい15〜20分。一度終われば、以降は「Part 2」を貼るだけでずっと更新できます。

> 🔑 **始める前に手元に用意するもの**
> - 共有GitHubアカウントの **ログイン情報**（メールアドレス／パスワード）※トークン発行で使います
> - Claudeのアカウント（Claude Code のログインで使用。お持ちのものでOK）

---

### A. ターミナルを開く

「ターミナル」は、Macに最初から入っている黒い画面のアプリです。

1. キーボードで **`command（⌘）＋ スペース`** を同時に押す（画面中央に検索窓が出る）
2. **`ターミナル`** と入力 → Enter
3. 白または黒い文字だけの小さなウィンドウが開けば成功

> 以降、「貼る」と書いてある部分は、この画面に貼り付けて Enter を押す、という意味です。
> （貼り付けは `command（⌘）＋ V`）

---

### B. Claude Code を入れる

1. 下の1行をコピーして、ターミナルに貼って Enter：
   ```
   curl -fsSL https://claude.ai/install.sh | bash
   ```
2. 文字がたくさん流れます。**1〜2分待つ**と止まります（これでインストール完了）
3. 続けて下を貼って Enter（Claude Code を起動）：
   ```
   claude
   ```
   - もし「command not found」と出たら、一度ターミナルを閉じて、A からもう一度開いて `claude` を試してください
4. 初回はログインを求められます。画面の案内どおりに進めると**ブラウザが開く**ので、Claudeアカウントでログイン → 「許可」
5. ターミナルに戻って、入力待ちの状態（カーソルが点滅）になれば成功

---

### C. サイトのデータを手元に取得する

Claude Code が起動している状態（B-5の画面）で、下をそのまま貼って Enter：

```
GitHubのリポジトリ mediaaidradio-lab/kachidoki-radio を
~/kachidoki-radio にクローンしてください。
```

「完了しました」のような返事が来れば成功です。
（これで社長のMacの中に、サイト一式のコピーが入りました）

---

### D. 公開用のカギ（アクセストークン）を発行する

これは「サイトを更新する権限のカギ」です。1本だけ作って、ずっと使い回します。

1. ブラウザで 👉 **https://github.com/settings/tokens/new** を開く
   - ログインを求められたら、**共有GitHubアカウント（mediaaidradio-lab）** のメール／パスワードでログイン
2. 開いた画面で、次のとおり設定：
   | 項目 | 入力・選択 |
   |---|---|
   | **Note** | `kachidoki-push`（メモ。何でもOK） |
   | **Expiration** | `No expiration`（一番下。無期限） |
   | **Select scopes** | 一番上の **`repo`** の四角にチェック（親をクリックすれば中身も全部入る） |
3. ページ一番下の緑のボタン **「Generate token」** を押す
4. `ghp_` から始まる長い文字列が表示されます
   - **この画面を閉じると二度と見られません。** すぐコピーして、**メモアプリ（メモ.app など）に貼って保存**してください
   - ⚠️ この文字列は「パスワードと同じ」。人に見せない・SNSやチャットに貼らない

これで準備はすべて完了です。お疲れさまでした！
（次回からは Part 2 を貼るだけ。Part 1 はもうやりません）

---

## ■ Part 2：毎回の更新（録音ができたら）

### 手順
1. 録音ファイル（m4a や mp3）を Mac の「ダウンロード」フォルダに入れる
2. Claude Code を起動し、下を貼る（**NN とタイトルとファイル名だけ書き換え**）：

```
新しいラジオ回を追加して公開してください。
録音ファイル: ~/Downloads/今日の録音.m4a
タイトル: 第NN回｜今日のテーマ

手順:
1. m4aならmp3に変換
2. episodeNN.mp3（連番）として配置
3. podcast.xml に <item> を追加
   （title / enclosure url / length=正確なバイトサイズ / pubDate=今日 / guid=kachidoki-ep-NNN）
4. git commit & push（pushでカギを聞かれたらユーザー名=mediaaidradio-lab、パスワード=保管したトークン ghp_…）
5. 反映URLを確認して教えて
```

3. 「反映URL」が返ってきたら成功。数分でサイトに、数時間〜半日でYouTube Musicに出ます

---

## ■ Part 3：覚えておくポイント

- **回数（NN）は連番**：前回が 03 なら次は 04。Claude Code に「今何回まである？」と聞けば教えてくれます
- **YouTube Musicは反映が遅い**：push後アプリに出るまで数時間〜半日（Google側の都合。サイトは正常なので待てばOK）
- **社員に配るURLは固定**：podcast.xml のURLは変わりません。一度配れば追加配布は不要
- **データは二重に安全**：GitHub と Mac の両方に同じものが残ります。Macの `kachidoki-radio` フォルダは消さないこと
